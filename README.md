# **C语言迷你植物花园模拟经营游戏开发任务清单**

## **一、项目概述**
**项目名称**：像素花园模拟器  
**项目类型**：命令行模拟经营游戏  
**核心玩法**：种植、照料、收获、销售各种植物，扩展花园规模，应对天气变化，成为园艺大师

## **二、游戏功能需求清单**

### **1. 数据管理系统 (必须实现)**
- [ ] **植物数据结构**
  - 至少包含：植物名称、生长阶段、生长值、健康值、所需水分/养分、总生长天数、出售价格
  - 定义3种以上不同植物（如雏菊、玫瑰、仙人掌）
  - 植物会每天消耗土地上的水分和养分
  
- [ ] **花园地块管理**
  - 使用数组或结构体数组管理多个地块
  - 每个地块记录：是否空置、种植的植物信息、已种植天数
  
- [ ] **玩家资源系统**
  - 金币（用于购买种子、升级工具）
  - 当前水分和养分值
  - 游戏天数/回合数
  - 工具等级（可选）

### **2. 核心游戏循环 (必须实现)**
- [ ] **主菜单界面**
  - 清晰显示当前游戏状态（天数、金币、资源、植物状态）
  - 提供至少5个操作选项
  - 简洁美观的ASCII边框或分隔线
  
- [ ] **天数推进机制**
  - 每次操作后或选择"下一天"时推进时间
  - 每天结束时自动更新植物状态

### **3. 游戏操作功能 (必须实现)**
- [ ] **种植功能**
  - 在空地块上选择植物种子进行种植
  - 消耗相应金币
  - 初始化植物状态
  
- [ ] **照料功能**
  - 浇水：增加植物水分，消耗玩家资源
  - 施肥：增加土壤养分，消耗玩家资源
  - 除虫：恢复植物健康值（随机触发虫害事件）

- [ ] **收获与销售**
  - 检测植物是否成熟（生长值达到100）
  - 出售成熟植物获得金币
  - 处理枯萎植物（铲除无收益）

### **4. 商店系统 (必须实现)**
- [ ] **种子购买**
  - 显示可购买的植物种子及其价格
  - 根据玩家金币判断是否可购买
  
- [ ] **道具购买**
  - 肥料：补充养分
  - 水壶升级：提高浇水效率（可选）
  - 花园扩展：增加新地块（可选）

### **5. 天气与随机系统 (必须实现)**
- [ ] **每日天气**
  - 至少3种天气类型（晴天、雨天、阴天）
  - 天气影响植物生长速度或资源消耗
  
- [ ] **随机事件**
  - 虫害事件：降低植物健康
  - 特殊事件：如"今日所有植物生长加快"
  - 使用随机数生成器控制事件概率

### **6. 数据持久化 (必须实现)**
- [ ] **游戏保存**
  - 将当前游戏状态保存到文件
  - 至少保存：玩家资源、所有植物状态、游戏天数
  
- [ ] **游戏加载**
  - 从文件读取游戏状态
  - 恢复到上次保存的游戏进度
  - 错误处理：文件不存在或损坏的情况

### **7. 用户界面与体验 (必须实现)**
- [ ] **状态显示**
  - 直观显示每个地块的植物状态
  - 使用简单的ASCII字符表示不同生长阶段
  - 重要信息（如可收获）需要突出显示
  
- [ ] **输入验证**
  - 防止无效输入导致程序崩溃
  - 提供清晰的错误提示
  - 重要操作前需要确认（如花费大量金币）

## **三、高级功能（加分项）**
### **扩展功能 (选择实现，加分)**
- [ ] **季节系统**
  - 每30天更换季节
  - 不同季节适合种植不同植物
  - 季节影响生长速度
  
- [ ] **成就系统**
  - 记录首次收获、连续照料、高额收入等成就
  - 成就解锁时显示祝贺信息
  
- [ ] **植物多样性**
  - 实现5种以上植物，各有不同特性
  - 稀有植物需要特定条件解锁
  
- [ ] **可视化界面**
  - 使用更丰富的ASCII图形表示植物和花园
  - 添加颜色支持（如果平台允许）
  
## **四、代码质量要求**
### **代码结构 (必须满足)**
- [ ] **模块化设计**
  - 合理划分函数，避免所有代码都在main函数中
  - 功能相似的函数组织在一起
  
- [ ] **头文件使用**
  - 使用.h文件声明结构体和函数原型
  - 合理组织多个源文件
  
- [ ] **命名规范**
  - 变量和函数使用有意义的英文名称
  - 避免使用拼音或缩写
  
- [ ] **代码注释**
  - 关键函数有功能说明
  - 复杂逻辑有行内注释
  - 文件开头有程序概述

### **程序健壮性 (必须满足)**
- [ ] **错误处理**
  - 文件操作失败时的处理
  - 内存分配失败的处理（如使用动态内存）
  - 用户输入错误的友好提示
  
- [ ] **边界检查**
  - 数组索引不越界
  - 数值范围合理（如金币不为负）

- **创意设计**：10分（独特的游戏机制）
- **文档质量**：5分（清晰的设计文档）

## **七、开发建议**
1. **分阶段开发**
   - 阶段1：搭建基础框架，实现数据结构和主循环
   - 阶段2：完成核心种植和照料功能
   - 阶段3：添加商店和天气系统
   - 阶段4：实现存档和高级功能
   - 阶段5：测试优化，完善文档

2. **测试要点**
   - 测试各种边界情况（金币为0时购买、地块已满时种植）
   - 测试存档/读档的完整性
   - 测试随机事件的平衡性

## **八、最后提醒**
- **从简单开始**：先实现最小可行版本，再逐步添加功能
- **经常测试**：每完成一个小功能就测试一次
- **代码备份**：定期备份代码，防止意外丢失
- **发挥创意**：在满足基本要求的基础上，让你的花园有特色
